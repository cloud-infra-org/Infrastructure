# Contributing to the Infrastructure Repository

Thank you for your interest in contributing to this repository.
Your support helps keep our infrastructure reliable and secure.

## Getting Started
1. Fork the repository and clone it to your local system.
2. Create a new branch for your changes.
3. Keep each branch focused on a single change.

## Contribution Guidelines
- Follow the existing folder structure and naming standards.
- Write clear and meaningful commit messages.
- Keep changes small and easy to review.
- Do not commit secrets, passwords, or sensitive data.

## Infrastructure Standards
- Validate all infrastructure changes before submitting.
- Format configuration files properly.
- Follow cloud, security, and cost-optimization best practices.

## Pull Request Process
1. Open a pull request with a clear description of your changes.
2. Explain why the change is required.
3. Make sure all checks pass successfully.
4. Address review comments before merging.

## Reporting Issues
- Use GitHub Issues to report bugs or suggest improvements.
- Provide clear details and steps to reproduce issues.

By contributing, you agree to follow these guidelines.
Thank you for helping improve our infrastructure.
