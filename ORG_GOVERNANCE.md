# Organization Governance

## Purpose
This organization is established to develop and manage software projects in a structured, secure, and collaborative manner. Our goal is to maintain code quality, reliability, and consistency across all repositories.

By defining clear roles and rules, we ensure smooth collaboration, accountability, and efficient software delivery.

## Roles and Responsibilities

### Admin
- Manage organization settings and security
- Add or remove members and assign roles
- Manage repositories and teams
- Define and enforce branch protection rules
- Handle policy and access issues

### Developer
- Write and maintain application code
- Follow coding and branching standards
- Create pull requests for all changes
- Review code and address feedback
- Fix bugs and issues

## Branch Management

- Only developers with approved write or maintain access are allowed to merge into main`
- All merges must be done through pull requests

### Who Can Change Branch Rules
- Organization admins only

## Rule Violations
- Minor violations may result in warnings
- Repeated or serious violations may lead to access removal
- Admin decisions are final
